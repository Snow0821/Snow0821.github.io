# Curriculum Map

This is the working tree for the class. It should stay visible and stable so
students can locate what they are learning.

## Operating Rules

- The tree shows location first; problem links are attached to the selected node.
- Keep BIKO/KOI official links as the primary contest-practice path.
- Use LeetCode as supplemental pattern practice, especially for basic data
  structures and common algorithm patterns.
- Keep each node small enough that students are not overwhelmed.
- Prefer collapsible branches over one long always-open list.

## 1. Foundations

- Programming basics
  - input/output
  - variables and types
  - conditionals
  - loops
  - functions
- Arrays and strings
  - indexing
  - counting
  - simple scans
  - boundary cases
- Complexity sense
  - O(1), O(N), O(N log N), O(N^2)
  - constraints to approach selection

## 2. Data Structures

- Linear structures
  - array
  - stack
  - queue
  - deque
  - hash table
  - prefix sum and difference array
- Trees
  - rooted tree
  - DFS order
  - subtree
  - LCA
  - tree DP
- Graphs
  - adjacency list
  - visited state
  - connected components
  - weighted graph
- Range data structures
  - Fenwick tree
  - segment tree
  - lazy propagation

## 3. Algorithms

- Brute force and simulation
- Sorting and binary search
- Two pointers and sliding window
- Greedy
- Dynamic programming
  - 1D DP
  - 2D DP
  - knapsack
  - interval DP
  - tree DP
- Graph algorithms
  - BFS and DFS
  - shortest path
  - topological sorting
  - MST
  - SCC
  - flow
- Strings and math
  - pattern matching
  - hashing
  - combinatorics
  - number theory

## 4. Problem-Solving Training

- Problem reading
  - constraints
  - input/output
  - hidden state
  - examples and edge cases
- Strategy selection
  - candidate approaches
  - complexity check
  - proof sketch
- Implementation planning
  - data representation
  - function split
  - test cases
- Debugging and upsolving
  - wrong answer analysis
  - time-limit analysis
  - alternative solution review

## 5. KOI Stages

- K0-K2: foundation and early implementation
- K3-K5: bridge problems across elementary, middle, and high divisions
- K6-K9: advanced data structures, graph, DP, and optimization
