# Memory Taxonomy

## Node Types

- `state`: current project status
- `decision`: confirmed choice and reason
- `question`: unresolved issue
- `topic`: reusable domain context
- `reference`: source, external material, or problem list
- `session`: session-specific work summary
- `template`: reusable document structure

## Status Values

- `active`: currently relevant
- `parked`: intentionally delayed
- `decided`: resolved and should guide future work
- `superseded`: kept only for history

## Topic Directory Pattern

Each major topic may contain:

- `overview.md`: stable summary
- `decisions.md`: topic-specific decisions
- `questions.md`: unresolved questions
- `notes-YYYY-MM-DD.md`: session-specific detail
- `references.md`: useful sources or problem links
